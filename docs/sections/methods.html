3  Methods

3.1 Research workflow

The project uses a GIS and remote-sensing workflow to compare green-space configuration and flood-related surface water change in Delft and Xi’an. The workflow consists of six main steps.

First, a 10 km by 10 km study area was selected for each city. Second, both study areas were divided into 100 m by 100 m grid cells. Third, binary green-space rasters were prepared for both cities and used to calculate green-space landscape metrics per grid cell. Fourth, before and after FWEI rasters were used to calculate surface water change. Fifth, the relationship between green-space metrics and FWEI-based indicators was explored using correlation analysis. Finally, K-means clustering was used to construct green-space typologies.

3.2 Spatial unit of analysis

A common grid-based method was used to make Delft and Xi’an comparable. Each study area was divided into 100 m by 100 m grid cells, resulting in 10,000 cells per city. This grid size was chosen as a compromise between spatial detail and computational feasibility. It is small enough to capture local variation in green-space configuration, but large enough to reduce noise from individual pixels.

All metrics were calculated per grid cell. This means that each cell received a set of green-space variables and flood-related variables. The grid therefore acts as the common analytical unit connecting the different raster datasets.

3.3 Data preparation

The main datasets used in the analysis were binary green-space rasters and FWEI rasters for Delft and Xi’an. The green-space rasters were prepared as binary maps, where green space has value 1 and non-green space has value 0. Because the original raster resolutions differed between the cities, both green rasters were aggregated to approximately 5 m resolution for the landscape metric calculation. Modal aggregation was used so that each new raster cell received the most common value from the original pixels.

The FWEI data consisted of before and after rasters for both cities. These rasters were aligned with the green-space analysis grid and used to calculate surface water change. The after raster was subtracted from the before raster using the following logic:

FWEI change = FWEI after - FWEI before

Positive values indicate an increase in the FWEI signal after the event, which is interpreted as an increase in surface water presence. This is treated as a flood-related surface water change proxy, not as direct flood depth.

3.4 Green-space metrics

Green-space configuration was measured using a combination of green coverage and patch-based landscape metrics. The decision to focus on patch metrics was made because the green rasters are binary maps with one main class of interest: green space. Patch metrics are therefore more suitable than broad class metrics because they describe the size, shape, and spatial arrangement of individual green patches.

The following green-space variables were calculated for each 100 m grid cell:

Variable Meaning Interpretation
green_percent Percentage of the grid cell covered by green pixels Total amount of green space
area Mean area of green patches Larger values indicate larger green patches
gyrate Mean radius of gyration of green patches Higher values indicate more spatially extensive patches
contig Mean contiguity of green patches Higher values indicate more internally connected patches
enn Mean Euclidean nearest-neighbour distance between patches Higher values indicate more isolated patches

The metrics were calculated in R using the landscapemetrics package. For each grid cell, the green raster was cropped and masked to the cell boundary. The metrics were then calculated only for the green class. If a cell contained no green pixels, the green metric values were set to zero or left as missing where appropriate.

3.5 FWEI change and flood-related variables

Two main FWEI-based variables were calculated per grid cell. The first was fwei_change_mean, which represents the mean FWEI change value within each grid cell. This variable captures the average change in surface water signal after the event.

The second variable was flood_share. To calculate this, the FWEI change raster was converted into a binary surface-water increase mask using a threshold. Pixels with FWEI change greater than 0.05 were classified as 1, while all other pixels were classified as 0. The mean value of this binary raster was then calculated per grid cell. This means that flood_share represents the proportion of each grid cell classified as FWEI-derived increased surface water.

This variable should be interpreted carefully. It does not represent flood depth and it does not prove that every classified pixel was flooded. Instead, it provides a consistent indicator of flood-related surface water increase for both cities.

3.6 Distance from flooded pixels to green space

An additional variable, mean_dist_green, was calculated to describe the spatial relationship between FWEI-derived flooded pixels and nearby green space. First, a distance raster was created from the binary green-space raster. This raster gives the distance from each non-green pixel to the nearest green pixel. The distance raster was then masked using the FWEI-derived binary surface-water increase mask.

For each grid cell, the mean distance from FWEI-derived flooded pixels to the nearest green pixel was calculated. Lower values indicate that the surface-water increase pixels are closer to green areas. Higher values indicate that the surface-water increase pixels are farther away from green areas.

3.7 Correlation analysis

Pearson correlation analysis was used to explore the relationship between green-space metrics and flood-related variables. The green-space variables were compared with fwei_change_mean and flood_share for both Delft and Xi’an.

The correlation analysis was used as an exploratory method. A positive correlation means that higher values of a green-space metric tend to occur in cells with higher FWEI change or flood share. A negative correlation means that higher values of a green-space metric tend to occur in cells with lower FWEI change or flood share. These correlations do not prove causality. They only show spatial association between variables.

The correlation outputs were saved as a CSV file and visualised as a correlation overview plot.

3.8 Typology construction

K-means clustering was used to construct green-space typologies. The clustering was based on the green-space metrics green_percent, area, gyrate, contig, and enn. These variables were standardised before clustering so that variables with larger numerical ranges did not dominate the clustering process.

Four clusters were created for each city. These clusters represent recurring types of green-space configuration. The typologies were then interpreted using the mean values of the green-space metrics and flood-related indicators.

The resulting typologies can be broadly interpreted as:

Typology General interpretation
Typology 1 Moderately green connected areas
Typology 2 Highly green and contiguous areas
Typology 3 Sparse green fragmented areas
Typology 4 Low green dispersed areas

The typologies were used to compare whether different green-space configurations show different levels of FWEI-based surface water change. The typology maps and summary tables were saved for both Delft and Xi’an.

3.9 Software and reproducibility

The analysis was carried out in R using a reproducible script-based workflow. The main packages used were terra, sf, landscapemetrics, dplyr, ggplot2, and readr. The scripts were stored in the src folder and all processed outputs were saved using relative paths.

The main scripts were:

Script Purpose
01_green_metrics_delft.R Calculates green metrics and FWEI-based variables for Delft
02_green_metrics_xian.R Calculates green metrics and FWEI-based variables for Xi’an
03_make_maps_and_plots.R Creates maps, correlations, typologies, and summary outputs

The final outputs include grid-level GeoPackages, CSV tables, correlation outputs, typology maps, and figure files. This structure makes the workflow easier to rerun and makes the analysis more transparent.
